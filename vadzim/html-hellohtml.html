HTML Tags

sind Elemente in der HTML-Datei, die mit einer spitzen Klammer anfangen.

Tags aus einem öffnenden Tagteil und einem schließenden Tagteil

werden gebraucht, um Inhalte zwischen diesen zwei Teilen vom Tag darzustellen. Man markiert den Anfang und das Ende für diese Inhalte. Inhalte von HTML-Tags sind Kinder von den Entsprechenden HTML-Elementen (HTML-Tags). Tags zum Rendern von Inhalten.

Tags, die nur aus einem Teil bestehen

sind nicht gedacht, um irgendwelche Inhalte wiederzugeben oder umzuschließen, sondern um eine Stelle zu markieren, wo ein HTML-Element(HTML-Tag) erscheinen soll

z.B br (break) fügt ein unsichtbares Element, welches die ganze Breite von Viewport (Bildschirm) annimmt und dadurch das nachfolgende Element dazu zwingt, in einer neuen Zeile zu erscheinen.


input-Elemente wiedergeben keine Inhalte, sondern sie warten auf die Inhalte (Eingabe) von dem Benutzer. Das ist der Grund, warum man den zweiten (schließenden) Tagteil nicht braucht.

hr (horizontal ruler) zeichnet eine Linie

img-Tag besteht nur aus einem Tail, weil nur der Anfangspunkt für ein Bild gesetzt wird. Die Anfangsgröße von diesem Bild hängt von der Bild-Datei ab, die durch das Attribut src verlinkt wird.

a-Tag verlinkt auch irgendwelche Inhalte, hat aber trotzdem einen schließenden Tag. Weil wir nicht die HyperReference (sprich: Verlinkung) und nicht die Webseite hinter dieser Verlinkung, sondern nur eine Beschreibung zeigen wollen. Quasi wohin diese Verlinkung führt.
